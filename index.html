GIT

Sistema de historial de cambios y control de versiones.

  • Repositorio: carpeta que guarda historial de cambios, puede ser local o remoto.
  • Commit: guardado del estado actual de archivos del repo y lo guarda como una versión específica. (fotografía)
    1. git add nombre-del-archivo ó
    2. git add .
    3. git status
    4. git commit -m "Mensaje descriptivo del commit"
    5. git origin main
  • Branch: línea de desarrollo separada dentro del repo, no afecta el código principal, luego se pueden fusionar las ramas.
  • Ramas: rama main es la principal, estable y probada y en ramas feature se prueban nuevas características.
  • Fusión (merge): combinación de cambios realizados en una sola versión.
  • Staging area: etapa intermedia, antes de realizar commit.
  • Directorio: carpeta.
  • git init: crea carpeta oculta llamada .git donde se almacenará toda la información del repositorio en el que se trabajará.

GITHUB

Plataforma en línea en el que se pueden guardar, compartir en proyectos de software.

  • Repositorio: lugar de almacenaje de código, documentación, y recursos. Puede ser público o privado.
  • Clonar repo (desde gitbash):
    1. Abrir directorio en donde se quiera guardar usando "ls" y "cd"
    2. git clone(enlace SSH)
    3. Para abrir en VSC: code .
  • git push: comando para subir cambios desde el repositio local, "git push origin main", para rama principal.
  • git pull: actualizar repositorio local de los cambios que han realizado otros colaboradores.
  • git fetch: descarga cambios para revisarlos antes de aplicarlos.
Github Pages

herramienta gratuita que ofrece GitHub para alojar sitios web estáticos directamente desde un repositorio.

  1. Repositorio de Github
  2. Pestaña "Settings"
  3. Sección "Pages"
  4. Apartado "Branch" elegir rama deseada y guardar en "Save"
  5. Luego de un momento designara el enlace.

Para mostrar en "<> code" dentro del repositorio.

  1. Settings en la sección About.
  2. Seleccionar "Use Your Github Pages website.
  3. Dar click en Save changes.

HTML

HyperText Markup Language

Lenguaje de marcado estándar utilizado para crear estructura del contenido de páginas web, permite mostrar títulos, texto, imagenes, enlaces, etc.

Las etiquetas se colocan entre <>

  • !DOCTYPEhtml: se agrega con "!", para seguir las reglas de HTML5
  • body: alberga el contenido visible de la página.
Etiquetas de texto
  • h1 a h6: etiquetas de encabezado.
  • p: párrafo de texto.
  • strong: negrita.
  • em: cursiva
Listas
  • ul: lista con puntos.
  • ol lista numerada..
  • li: elemento de la lista.
Enlaces
  • a: crea enlaces en HTML.
  • href: url del enlace
Imágenes
  • img: inserta imágenes.
  • src: ruta de la imagen-
  • alt: texto para describir imagen.
  • title: muestra un texto flotante cuando se coloca el cursor sobre la imagen.
Estructura semantica
  • header: cabecera de la página o de una sección, puede contener logotipos, menús, título.
  • nav: sección de una página cuyo propósito es proporcionar enlaces de navegación, ya sea dentro del documento actual o hacia otros documentos.
  • main: contenido principal del documento, único.
  • footer: pie de página.
  • article: representa una composición autónoma en un documento, página, aplicación o sitio, que está destinada a ser distribuida o reutilizada.
  • section: representa una sección genérica e independiente de un documento, que no tiene un elemento semántico más específico que la represente.
Divisiones y contenedores
  • div: contenedor de bloque que se utiliza para agrupar elementos o secciones, empieza en una nueva línea y ocupa el ancho disponible.
  • span: es un contenedor en línea, que se usa para agrupar pequeños fragmentos de texto u otros elementos dentro de una línea.
FORMULARIO

Herramienta que permite que el usuario envie datos al servidor.

Elementos dentro de <>

  • form: contenedor principal de todos los elementos del formulario.
  • input: permite aceptar datos del usuario, hay una amplia variedad de tipos de datos de entrada y widgets de control disponibles. El tipo de dato que el usuario puede ingresar se define mediante el atributo type.
  • label: describe el propósito de un campo de entrada, generalmente se asocia con input para mejorar la accesibilidad y el usuario identifique la información que se debe proporcionar.
  • textarea:recoger grandes cantidades de texto, como comentarios o descripciones. A diferencia de input, que generalmente es una sola línea
  • button: botones iteractivos dentro del formulario, se pueden modificar de acuerdo a las necesidades.
Campos de texto
  • input type="text": entrada versatil, campos de texto básicos de una sola línea como nombre o dirección.
  • input type="email": entrada para ingresar y editar dirección de correo electrónico, puede ser validad por el mismo nevegador.
  • input type="password": entrada para contraseñas en la que enmascaran el texto.
Botones de envío
  • button type="submit": El botón envía datos del formulario cuando se da click y permite modificar el botón.
  • input type="submit": Envía datos del formualrio, pero no permite contenido adicional dentro del botón.
Etiquetas de agrupacióm:
  • fieldset: se utiliza para agrupar varios controles y etiquetas ( ) dentro de un formulario.
  • legend: proporciona un título o descripción para el grupo de campos dentro de un fieldset.

CSS

Cascading Style Sheets (hojas de estilo en cascada)

Leguaje que se usa para estilizar una página web.

Regla CSS

  1. Selector: parte o partes (se puede hacer combinaciones) del archivo HTML al que se le aplicarán los estilos, luego del selector se abre llave {.
    • Elemento, por ejemplo: h1, p, img
    • Clase o clases, por ejemplo: .clase
    • ID, por ejemplo: #id
    • Selector universal: * en el que se aplica un estilo a todo el proyecto.
  2. Declaración: precisa a cuál de las propiedades se estilizará. (Propiedad: propiedad-valor);
  3. Propiedad: característica que se le modificará al selector.
  4. Valor de propiedad: color, cantidad, estilo específico de la propiedad elegida.
  5. Al finalizar regla se cierra }.
  • index.html: nombre que se le asignará al archivo HTML para poder funcionar correctamente.
  • styles.css: nombre que se le dará a archivo CSS dentro del mismo repositorio.

Para vinvular archivo CSS al HTML colocar entre <> link rel="stylesheet" href="styles.css" dentro de la etiqueta head.

Pseudoclases

Aplican estilos a ciertos estados de un elemento.

  • :hover; cambia el estilo de un elementos cuando pasa el mouse sobre él.
Pseudoelementos
  • Aplican estilos a una parte concreta de un elemento, por ejemplo: seleccionar primera línea de un texto.
Display
  1. display: inline; elementos que se alinean en la misma línea, llegan al final y saltan al siguiente espacio.
  2. display: block; elementos que ocupan todo el ancho disponible y comienzan en una nueva línea, se le spuede dar un tamaño en específico.
  3. display: inline block; combinación de displays anteriores.
  4. display: flex; elementos se alinean de distintos formas y distribuir espacio entre ellos.
  5. display: grid; rejilla que permite coloacr elementos en filas, columnas, se puede definir la cantidad, tamaño, distancia entre ellas, etc.
  6. display: none; desaparece elementos por completo
Propiedades
  • float: controla como se alinean los elementos dentro de un contenedor.
  • Position: controla como se posiciona un elemento HTML en relacioón con su contenedor.
    1. position: absolute; se posiciona con respecto al contenedor padre, al usar esta propieda se puede usar (top, right, bottom y left)
    2. position: relative; elementos se queda en el flujo de la página (top, right, bottom y left)
    3. position: fixed; elementos se posicionan de acuerdo a absolute y relative (top, right, bottom y left)
    4. position: sticky; similar e relative sirve para que un elemento se quede fijo en su posición al hacer scroll, pero solo dentro de su contenedor.
  • z-index: sirve para controlar qué elemento se coloca "encima" o "debajo" de otros cuando se superponen. Un número más alto lo pone más arriba. Solo tiene efecto sobre elementos que están posicionados (es decir, con position: relative, absolute, fixed, o sticky).
  • color: cambia color de los elementos.
  • background-color: fondo de color.
  • font-siza: tamaño de fuente
  • font-familiy: fuente (se puede agregar fuentes desde Google Fonts).
  • text-align: texto alineado.
Modelo de caja

Cada elemento dentro de un proyecto de HTML y CSS en el navegador se muestra en una caja rectangular. Cada elemento es una caja dentro de otra caja.

Área de elementos

  • Content: tamaño total en px del tamaño del elemento.
  • Padding: establece el área de relleno en los cuatro lados de un elemento a la vez.
  • Border: perimetro que rodea al elemento y al padding.
  • Margin: margen de los cuatros lados (propiedades: margin-top, margin-right, margin-bottom y margin-left)
  • width y height: ancho y altura de la caja.

Box-sizing

  • box-sizing: content-box; valor por defecto, ancho y altura de un elemento se aplican ala caja sin contar relleno ni borde.
  • box-sizing: border-box; el tamaño del elemento se simplifica, ancho y altura que se define incluyen relleno y bordes.
Flexbox

Permite acomodar la disposición de elementos de manera flexible, ya sea horizontal o vertical.

  • Cuando se aplica display:flex los elementos hijos se comportan con reglas de flexbox.
  • flex-direction; dirección del eje principal
    1. row; alinea elementos de izquierda a derecha.
    2. row-reverse: alinea elementos de derecha a izquierda.
    3. column: alinea elementos de arriba a abajo.
    4. column-reverse: alinea elementos de abajo a arriba.
    5. flex-start; elementos se alinean al inicio del contenedor.
    6. flex-end; elementos se alinean al final del contenedor.
    7. center; elementos se alinean al centro del contenedor.
    8. space-between; mantiene el mayor espacio entre elementos.
    9. space-around; mantiene espacio entre los elementos y también al principio y al final.
  • justify-content: alienación en el eje principal.
  • align-items: alienación eje cruzado
  • align-content: controla cómo se distribuyen las líneas de elementos a lo largo del eje cruzado.
  • flex-wrap: permite que los elementos se distribuyan en múltiples líneas cuando el espacio disponible sea insuficiente.
    1. nowrap: elementos se colocan en una sola línea
    2. wrap: elementos se acomoden en varias lineas si no saven en una fila o una columna.
    3. wrap-reverse: similar a wrap, pero la línea nueva se coloca por encima de la línea original.
  • flex-grow: controla cuánto puede crecer un elemento en relación con los demás.
